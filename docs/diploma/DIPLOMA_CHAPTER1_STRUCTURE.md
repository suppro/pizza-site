## 1.4. Функциональные требования к системе

### 1.4.1. Требования к модулю каталога товаров
- Иерархическая структура категорий и подкатегорий
- Интеллектуальный поиск по названию, артикулу, описанию
- Система фильтрации (по категориям, цене, наличию)
- Отображение карточек товаров с фотографиями
- Подробные описания и технические характеристики
- Отображение актуального наличия и цен

### 1.4.2. Требования к модулю корзины и оформления заказа
- Добавление товаров в корзину с указанием количества
- Многошаговый процесс оформления заказа
- Выбор способа доставки (самовывоз, курьер, транспортная компания)
- Выбор способа оплаты (наличные, карта, банковский перевод)
- Личный кабинет для зарегистрированных клиентов
- История заказов

### 1.4.3. Требования к модулю управления заказами
- Система статусов заказов (новый, в обработке, выполнен, отменен)
- Автоматическая отправка email-уведомлений при изменении статуса
- Интеграция с системой 1С для автоматической выгрузки заказов
- Отслеживание статуса синхронизации с 1С

### 1.4.4. Требования к административной панели
- Управление товарами (добавление, редактирование, удаление)
- Управление категориями (создание иерархической структуры)
- Просмотр и обработка заказов
- Изменение статусов заказов
- Базовая аналитика по продажам
- Управление пользователями системы

## Примеры содержания разделов

### 1.1.1. Характеристика деятельности АО «Арвиай»
- История и основные направления деятельности
- Организационная структура
- Ассортимент продукции (примеры категорий: двигательные компоненты, шасси и тормоза, электроника и авионика)
- Особенности работы с клиентами (оптовые и розничные продажи)
- Текущие каналы продаж

### 1.2.1. Обзор платформ для создания интернет-магазинов
- **Готовые решения:**
  - OpenCart (преимущества: простота, недостатки: ограниченная кастомизация)
  - WooCommerce (преимущества: интеграция с WordPress, недостатки: зависимость от CMS)
  - PrestaShop (преимущества: функциональность, недостатки: сложность настройки)
- **Фреймворки для разработки:**
  - Laravel (выбор для проекта)
  - Symfony
  - Yii2
- **Сравнительная таблица** по критериям: функциональность, производительность, стоимость, поддержка

### 1.4.1. Требования к модулю каталога товаров
- Иерархическая структура категорий (пример: Двигательные компоненты → Компрессоры → Осевые компрессоры)
- Поиск по полям: название, артикул (SKU), описание
- Фильтры: категория, цена (от-до), наличие на складе
- Карточка товара должна содержать:
  - Фотографии (основная и дополнительные)
  - Название и артикул
  - Описание
  - Технические характеристики (JSON-формат для гибкости)
  - Цена
  - Количество на складе
  - Кнопка добавления в корзину

### 1.6.1. Обоснование выбора Laravel
- **Преимущества:**
  - Активная разработка и поддержка
  - Богатая экосистема пакетов
  - Встроенная система аутентификации
  - ORM Eloquent для работы с БД
  - Blade-шаблонизатор
  - Миграции БД
  - Система очередей для фоновых задач
  - Встроенная система уведомлений
- **Соответствие требованиям проекта:**
  - Быстрая разработка
  - Безопасность из коробки
  - Возможность интеграции с 1С через REST API
  - Масштабируемость
