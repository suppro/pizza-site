# 1.5. Выводы по главе

В первой главе дипломной работы проведен комплексный анализ предметной области, существующих решений и определены требования к разрабатываемой системе. На основе проведенного анализа сформулированы цели и задачи проекта, выбран технологический стек для реализации интернет-магазина АО «Арвиай».

## Основные результаты анализа

Проведенный анализ деятельности АО «Арвиай» показал необходимость автоматизации бизнес-процессов компании. Выявлены следующие основные проблемы текущей системы работы: высокая операционная нагрузка на менеджеров, риск ошибок при ручном вводе данных, ограниченная доступность информации для клиентов, неэффективное использование рабочего времени и сложность масштабирования бизнеса [4; 16].

Анализ существующих решений на рынке показал, что для проекта наиболее подходящим является подход самостоятельной разработки на основе фреймворка Laravel. Данный выбор обеспечивает полный контроль над архитектурой системы, возможность реализации специфических требований проекта (работа с техническими характеристиками в формате JSON, интеграция с 1С:Предприятие), высокую гибкость и масштабируемость [11; 14; 20].

Для интеграции с системой 1С:Предприятие выбран метод REST API через HTTP-сервисы, который обеспечивает простоту реализации, высокую производительность, масштабируемость и возможность обмена данными в реальном времени.

## Определенные требования

На основе проведенного анализа сформулирована основная цель проекта: разработка и внедрение современного интернет-магазина для АО «Арвиай», направленного на увеличение объема продаж и повышение эффективности работы с клиентами.

Определены функциональные требования к основным модулям системы:
- Модуль каталога товаров с иерархической структурой категорий, интеллектуальным поиском и системой фильтрации
- Модуль корзины и оформления заказа с трехшаговым процессом оформления
- Модуль управления заказами с системой статусов, автоматической отправкой уведомлений и интеграцией с 1С
- Административная панель для управления товарами, категориями, заказами и пользователями

Определены нефункциональные требования: производительность (время загрузки страниц не более 2 секунд), безопасность (защита от SQL-инъекций, XSS-атак, CSRF-атак), надежность, удобство использования (адаптивный дизайн), масштабируемость и требования к сопровождению.

## Выбранный технологический стек

Выбран следующий технологический стек:
- **Backend**: Laravel 12, PHP 8.2+
- **База данных**: MySQL 8.0 (с поддержкой JSON)
- **Frontend**: Blade, Tailwind CSS, JavaScript, Vite
- **Инструменты**: Git, Composer, Guzzle HTTP Client

Выбранный технологический стек полностью соответствует требованиям проекта и обеспечивает эффективную разработку, высокую производительность, безопасность и масштабируемость системы.

## Ожидаемые результаты

Реализация проекта позволит достичь следующих результатов:
- Снижение операционной нагрузки на менеджеров на 40-50%
- Сокращение времени обработки заказа с нескольких часов до нескольких минут
- Повышение удовлетворенности клиентов за счет возможности самостоятельного оформления заказов
- Снижение количества ошибок при обработке заказов
- Возможность масштабирования бизнеса без пропорционального увеличения операционных расходов

## Заключение

Проведенный анализ показал необходимость и обоснованность создания интернет-магазина для АО «Арвиай». Выбранный подход к разработке и технологический стек обеспечивают возможность создания системы, полностью соответствующей требованиям проекта. Определенные функциональные и нефункциональные требования служат основой для проектирования и разработки системы, что будет рассмотрено во второй главе дипломной работы.
